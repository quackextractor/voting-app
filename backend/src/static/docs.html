6. API Endpoints

Method URL Endpoint Description and request/response body
GET /health Basic system health check for external monitoring services.
Response: { "status": "ok", "totalVotes": 5, ... }
GET /api/stats Detailed system, memory, and application statistics for the monitor dashboard.
Response: { "uptime": {...}, "memory": {...}, "application": {...} }
GET /api/results Returns current voting states. No authorization is required.
Response: { "options": [{ "id": "a", "votes": 15 }, ...] }
POST /api/vote Saves a new user vote. Security: Double-voting is prevented using IP tracking and persistent client cookies.
Body: { "optionId": "b" }
Response: { "success": true, "options": [...] }
POST /api/reset Resets all votes if the token matches the server configuration variable.
Body: { "token": "super_secret_token" }
Success response: { "success": true, "message": "Poll reset" }
Error response: { "success": false, "message": "Invalid token" } (Status 403)
