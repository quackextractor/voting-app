### [#2 Display Warning When User Attempts to Vote Again (UX/UI)](https://github.com/quackextractor/voting-app/issues/2)
**Author:** Anh Vu (VietnameseDelegation)  
**Created:** 2026-03-07T09:37:58Z  
**State:** OPEN  

The poll allows users who have already voted to submit another vote without displaying any warning or notification that they have already participated. This creates confusion because the user can interact with the poll again, but the vote has no effect.

#### Reproduction Steps

1. Visit the website containing the poll.
2. Vote in the poll.
3. Leave the page or return to the website later.
4. Open the same poll again.
5. Select an option and attempt to vote again.

#### Result
The poll appears interactive and allows the user to select and submit an option again, but the vote is not counted because the user has already voted. However, the system does not display any warning, notification, or message informing the user that they have already voted. A clear message or pop-up should be shown to prevent confusion and improve user experience.

### [#1 Allow users to see what they voted for](https://github.com/quackextractor/voting-app/issues/1)
**Author:** Anh Vu (VietnameseDelegation)  
**Created:** 2026-03-07T09:34:44Z  
**State:** OPEN  

Users cannot see which option they selected after submitting their vote in the poll. The poll results are visible, but the user’s own choice is not indicated, which can cause confusion about what they voted for.

#### Reproduction Steps
Visit the website containing the poll.
Select an option and submit your vote.
View the poll results after voting.

#### Result
The poll displays the results but does not indicate which option the user voted for. The user should be able to clearly see their selected option (for example, highlighted or labeled as “Your vote”) since the vote must already be stored in the system.
